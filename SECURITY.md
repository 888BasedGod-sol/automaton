# Security Policy

## Supported Versions

Security fixes are applied to the latest main branch.

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Instead, report privately to the maintainers with:

- A clear description of the issue
- Reproduction steps or proof of concept
- Impact assessment
- Suggested remediation (if available)

You can use GitHub Security Advisories (preferred) or contact the maintainer directly.

## Sensitive Data Guidelines

- Never commit private keys, wallet secrets, API tokens, or `.env` files.
- Rotate any credential immediately if exposure is suspected.
- Assume any committed secret is compromised.
